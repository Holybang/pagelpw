详细描述

该接口类表示样本数据库，用于训练和测试样本的组织和管理。

样本数据库支持两种类型：

  • 持久化数据库：样本文件和数据库信息将被序列化保存到硬盘上的数据库工作目录位置。当数据库内容发生变化时，如添加新样本、删除样本等，将自动进行保存。
  • 内存数据库：数据库依靠主存进行数据存储，其样本文件和数据库信息不保存在硬盘上。因此当应用程序重启时，将丢失该数据库的所有信息。

使用这个接口，需要创建一个 LSampleDatabase 对象。

示例代码

C++
// create "mylabel" sample database
ILSampleDatabasePtr sampleDb(__uuidof(LSampleDatabase));
sampleDb->Create(L"mylabel", L"d:/test/mylabel");
// add sample images
sampleDb->AddSampleImage(LPVSampleType::LPVSampleTrain, img1, region1);
sampleDb->AddSampleImage(LPVSampleType::LPVSampleTest, img2, region2);
// load "anotherlabel" sample database
ILSampleDatabasePtr sampleDb2(__uuidof(LSampleDatabase));
sampleDb2->Load(L"d:/test/anotherlabel");
C#
// create "mylabel" sample database
LSampleDatabase sampleDb = new LSampleDatabase();
sampleDb.Create("mylabel", "d:/test/mylabel");
// add sample images
sampleDb.AddSampleImage(LPVSampleType.LPVSampleTrain, img1, region1);
sampleDb.AddSampleImage(LPVSampleType.LPVSampleTest, img2, region2);
// load "anotherlabel" sample database
LSampleDatabase sampleDb2 = new LSampleDatabase();
sampleDb2.Load("d:/test/anotherlabel");

◆ Refresh()

LPVErrorCode Refresh ( )

手动刷新整个数据库。对于内存数据库，该动作无实际效果。 对于持久化数据库，将重新扫描所有样本文件夹，然后重新加载所有样本。不可访问的样本和样本文件夹将被忽略和跳过。
该动作比较耗时，不推荐高频次调用。

返回值
