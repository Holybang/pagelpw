Properties

int MaxCount [get, set]
 The maximum number of the images, in 0 ~ 256.
For example, if set to 16, it means we maintain a image cache of 16 size in the list.
If more than the setting number of images are pushed into the list, the earlier ones will be popped.
By default, it's 0, means no limitation.
 

Detailed Description

This interface defines a list of LImage object.

It maintains a cache of a pre-defined size to store grabbed frames, then used in multi-frame blending, HRD or image stitching tasks.

To use this interface, you should create a LImageList object.

Example Code

C++
ILImageListPtr m_img_list(__uuidof(LImageList));
m_img_list->MaxCount = 3;
m_img_list->Add(m_frame1);
m_img_list->Add(m_frame2);
m_img_list->Add(m_frame3);
m_img_list->Add(m_frame4); // frame 1 is popped
C#
LImageList m_img_list = new LImageList();
m_img_list.MaxCount = 3;
m_img_list.Add(m_frame1);
m_img_list.Add(m_frame2);
m_img_list.Add(m_frame3);
m_img_list.Add(m_frame4); // frame 1 is popped
