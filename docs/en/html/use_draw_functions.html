Draw Functions

For the image and the drawable LPV classes which implements the ILDrawable interface, we usually provide the Draw functions to paint the class's content on the provided device context, aka. the HDC handle.

It is a more flexible and light-weighted API for the users to fully control the graphics rendering of their applications, compared with the LDisplay Control.

Here are some example draw functions for the image, region and detection results:

void ILImage::Draw (HDC hdc, double zoomX, double zoomY, double panX, double panY);
void ILRegion::Draw (HDC hdc, BOOL bHandles, double zoomX, double zoomY, double panX, double panY);
void ILMatchResults::Draw (HDC hdc, int index, LPVPatDrawFlags drawFlags, double zoomX, double zoomY, double panX, double panY)
void ILMatchResults::DrawAll (HDC hdc, LPVPatDrawFlags drawFlags, double zoomX, double zoomY, double panX, double panY)

They have the common parameters for specifying the drawing device context and controlling the view port.

Parameters
[in]hdcHandle to the device context of the destination window or bitmap.
[in]zoomXFactor for zooming in or out in the x-coordinate direction. By default, it's in the true scale.
[in]zoomYFactor for zooming in or out in the y-coordinate direction. Setting to 0 means same zoom factor as zoomX, aka. isotropic scaling.
[in]panXtranslation in the x-coordinate direction for panning. The final display x position is calculated as: \( (x + panX) \times zoomX \)
[in]panYtranslation in the y-coordinate direction for panning. The final display y position is calculated as: \( (y + panY) \times zoomY \)

They may also have the extra parameters to control drawing behavior.

Examples for Use the Draw Functions

Draw on the Bitmap

C++
void DrawImageAndResultsOnStatic(CStatic* pic_item, ILImagePtr img, ILRectRegionPtr roi, ILMatchResultsPtr matchResults)
{
// you should firstly modify the static control's style to accept bitmap
// you should do it in the dialog's initialization method OnInitDialog(), like:
// ((CStatic*)GetDlgItem(IDC_IMG_PIC))->ModifyStyle(0xF, SS_BITMAP, SWP_NOSIZE);
CRect picRect;
pic_item->GetClientRect(picRect);
// adapt to size of ui widget
double zoomx = (double)picRect.Width() / img->Width;
double zoomy = (double)picRect.Height() / img->Height;
CDC* picdc = pic_item->GetDC();
if (!picdc) return;
// create the new memory DC and bitmap for painting
HDC pichdc = picdc->GetSafeHdc();
HDC hdc = CreateCompatibleDC(pichdc);
HBITMAP bitmap = CreateCompatibleBitmap(pichdc, picRect.Width(),
picRect.Height());
// replace with new created bitmap
HBITMAP prevBitmap = (HBITMAP)SelectObject(hdc, bitmap);
if (prevBitmap)
DeleteObject(prevBitmap);
// white background
FillRect(hdc, &picRect, (HBRUSH)GetStockObject(WHITE_BRUSH));
// call draw function to draw the image and the match results on the bitmap
// use 0, 0 panning
img->Draw(hdc, zoomx, zoomy, 0, 0);
roi->Draw(hdc, VARIANT_TRUE, zoomx, zoomy, 0, 0);
matchResults->DrawAll(hdc, LPVPatDrawDefault, zoomx, zoomy, 0, 0);
// refresh the ui
BitBlt(pichdc, picRect.left, picRect.top, picRect.Width(), picRect.Height(),
hdc, 0, 0, SRCCOPY);
pic_item->ReleaseDC(picdc);
// release the memory hdc and bitmap
DeleteObject(hdc);
DeleteObject(bitmap);
}
C#
void drawImageAndResultsOnStatic(PictureBox pic_item, LImage img, LRectRegion roi, LMatchResults match_results)
{
// adapt to size of ui widget
double zoomX = (double)pic_item.Width / img.Width;
double zoomY = (double)pic_item.Height / img.Height;
// create the new memory DC and bitmap for painting
Bitmap bmp = new Bitmap(pic_item.Width, pic_item.Height);
Graphics gfx = Graphics.FromImage(bmp);
gfx.Clear(Color.White); // white background
IntPtr hdc = gfx.GetHdc(); // get hdc for drawing
// call draw function to draw the image and the match results on the bitmap
// use 0, 0 panning
img.Draw(hdc, zoomX, zoomY);
roi.Draw(hdc, true, zoomX, zoomY);
match_results.DrawAll(hdc,
LPVPatDrawFlags.LPVPatDrawDefault,
zoomX, zoomY);
gfx.ReleaseHdc(hdc);
gfx.Dispose();
// display bitmap in picture control
if (pic_item.Image != null)
pic_item.Image.Dispose();
pic_item.Image = bmp;
}

void CMFCDlg::OnPaint()
{
// balabala, other painting logic
CPaintDC dc(this); // device context for painting
HDC hdc = dc.GetSafeHdc();
CRect picRect;
GetClientRect(picRect);
// adapt to size of ui widget
double zoomx = (double)picRect.Width() / m_img->Width;
double zoomy = (double)picRect.Height() / m_img->Height;

// call draw function to draw the image and the match results on the bitmap
// use 0, 0 panning
m_img->Draw(hdc, zoomx, zoomy, 0, 0);
m_roi->Draw(hdc, VARIANT_TRUE, zoomx, zoomy, 0, 0);
m_matchResults->DrawAll(hdc, LPVPatDrawDefault, zoomx, zoomy, 0, 0);
// balabala, other painting logic
CDialogEx::OnPaint();
// Note, when the things you want to draw changed:
// you may call Invalidate() to request a delayed repaint by the message loop system.
// you may call UpdateWindow() to force a immediate repaint.
// you may use some flags to control which LPV objects should be drawn.
// you may use double-buffering technique to avoid flickering,
// by painting the objects on an in-memory DC first then copying the entire canvas to the target DC.
}
